Redux



Redux uses 3 constructions to manage and manipulate applicant's state:.

  • Action
  • Reducer
  • Store

    • Actions

  • Everytime your application wants to change the state of the app, it should dispatch an action
  • For example, taking the Trello application, adding a new card to a list would be a single action.
  • These actions are then handled by a reducer, which actually makes the changes to the state.
  • Actions are represented by simple objects that have a type property
  • The type property's value is a string indicating the type of action being carried out.
  • Any additional properties are known as the action's payload

    • Reducers

  • Reducers are responsible for modifying the application state based on an action.

    • A Reducer takes two Arguments

  • 1. The first is the current state
  • 2. and the second is an action
