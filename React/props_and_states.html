Props

    • defaultProps

  • Often you want a prop to have a default value that is used if an alternative isn't supplied when rendering the component.
  • This is achieved using the defaultProps attribute.
  • In the Card component, if a text prop isn't provided then the component will fall back to using an empty string.

    • Validating props with prop-types

  • If you need extra confirmation that you are passing the correct information to a component via its props then you can use the prop-types library.
  • There are a wide range of different validators which cover all of the core JavaScript types; you can see a full list of these in the prop-types documentation.

    • States

  • The state is an object that is stored as part of a component and can be updated by the component itself.
  • only stateful components can contain state. Stateless components can only be manipulated by changing the props passed to them.
  • Stateful components are more powerful, but stateless components are often more reusable and simpler to reason about.
  • This is the fundamental trade-off between stateless and stateful components. Stateful components are more powerful,
  • but stateless components are often more reusable and simpler to reason about.
  • As a general rule, you should prefer stateless components and only create stateful components when you really need to.
