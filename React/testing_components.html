TESTING COMPONENTS



  • Enzyme Enzyme is a library that lets you render a React component and make assertions about its output and behavior.

    • What should you test

      3 main categories
  • Smoke Test: Do your components render something without falling over? These are quick sanity checks that your code is at least capable of rendering something.
  • Props and state: Does what's rendered reflect the component's props and state?
  • Callbacks and events: Are callbacks fired when the user interacts with a component? Is the state updated correctly?

    • Structure of the tests

    When you run npm test, Create React App invokes a test runner called Jest

    Jest is similar to mocha. Searches for tests and runs them

    Jest looks for the files matching the following naming conventions

  • Files with .js suffix in __tests__ folders.
  • Files with .test.js suffix.
  • Files with .spec.js suffix.

    • Test Structure

  • You have a describe block, which is used to group your test cases.
  • The component name is used here to clearly identify which component is being tested.
  • Inside the describe block you have a series of it blocks.

    • Shallow Rendering

  • The shallow function uses a technique called shallow rendering.
  • Any child components aren't expanded, and their output cannot be inspected. You can however look at which child components were rendered, and inspect the props passed to them
  • So for example, if AddForm rendered a child component called AddFormButton, our smoke test would make sure that no error was thrown within AddForm's render method, but AddFormButton could still be throwing and our test wouldn't account for this. You would need a separate smoke test for AddFormButton to make sure that the entire component rendered correctly.

    • Full DOM rendering

    In full DOM rendering, the entire component including its children are rendered into an in-memory DOM

    If the feature you are trying to test uses any of the following features or techniques, then you will need to use full DOM rendering rather than shallow rendering:

  • Refs
  • Life cycle methods
  • Directly interacting with DOM APIs

    • Enzyme's full DOM rendering works almost identically to its shallow rendering but uses the mount method rather than the shallow method.

    The API is largely identical, aside from some small changes to accommodate differences between shallow and full DOM rendering.
